# 鹰翼外部靶场 AI 攻防工作台

## 宣传使用手册

适用对象：蓝队队员、教员、裁判人员  
适用场景：网络攻防比赛、蓝队攻防演练、外部临时靶场测试、教学演示

---

## 一、平台简介

**鹰翼外部靶场 AI 攻防工作台** 是一套面向蓝队队员的 AI 辅助攻防演练平台。

在比赛或训练中，主办方通常会临时给出一批外部靶场目标，例如 IP、网段、域名或 URL。队员需要在有限时间内完成信息收集、漏洞发现、漏洞验证、权限获取和 writeup 编写。

本平台的作用，就是帮助蓝队队员把这些流程变得更快、更清晰、更规范。

平台可以自动完成：

```text
目标导入
资产发现
端口扫描
服务识别
漏洞扫描
风险排序
并行渗透验证
证据采集
报告生成
容器清理
```

队员只需要根据平台给出的结果，人工选择要深入验证的漏洞，平台会为每个验证任务自动创建临时沙箱容器，并行执行，结束后自动回收资源。

## 二、平台能解决什么问题

传统比赛中，队员经常遇到这些问题：

```text
目标多，不知道先看哪个
扫描结果杂乱，难以判断优先级
多个漏洞无法并行验证
工具环境不统一
过程证据容易丢
writeup 后期整理困难
裁判难以复核过程
```

鹰翼工作台可以帮助解决：

```text
自动整理资产和漏洞
自动给出风险排序
支持多个漏洞并行验证
每次验证使用独立临时容器
自动保存截图、日志、请求响应和结果
项目结束自动生成整体渗透报告
裁判可查看任务过程、容器记录和证据链
```

## 四、平台核心特点

### 1. 面向外部临时靶场

平台不要求提前搭建本地靶场。

比赛开始后，只需要导入主办方给出的目标，例如：

```text
10.10.10.0/24
192.168.56.101
http://target.example
https://app.range.local
```

即可开始工作。

### 2. AI 辅助漏洞排序

平台会结合端口、服务、漏洞类型、暴露面、置信度等因素，自动给出突破口优先级。

示例：

| 排名 | 目标 | 发现 | 风险 | 建议 |
|---|---|---|---|---|
| 1 | 10.10.10.5:8080 | Web 管理后台暴露 | 高 | 优先深度验证 |
| 2 | 10.10.10.8:445 | SMB 服务开放 | 中高 | 检查配置风险 |
| 3 | app.target.local | Web 漏洞模板命中 | 中 | 人工复核 |

队员不需要从大量原始日志中一点点翻找，平台会先把重点整理出来。

### 3. 人工确认后再深入验证

平台不会盲目自动攻击。

流程是：

```text
平台发现漏洞
-> 平台排序推荐
-> 队员人工选择
-> 平台启动深度验证
-> 保存证据
```

这样既提高效率，也能避免不必要的高风险操作。

### 4. 支持并行渗透验证

如果同时发现多个可疑漏洞，队员可以勾选多个目标，一键启动并行验证。

例如：

```text
漏洞 A：Web 管理后台弱配置
漏洞 B：中间件版本风险
漏洞 C：目录暴露
漏洞 D：接口未授权访问
```

平台会为这些任务分别启动临时容器，同时执行验证，提高比赛效率。

### 5. 每次渗透使用临时容器

每个验证任务都会在独立沙箱容器中运行。

好处是：

```text
工具环境统一
任务之间互不影响
失败任务不会污染系统
方便记录完整过程
结束后自动释放资源
```

任务完成后，平台会自动收集：

```text
执行结果
命令日志
标准输出
错误输出
截图
HTTP 请求响应
证明文件
时间线
```

然后释放容器。

### 6. 自动生成整体报告

项目结束后，平台会把所有结果汇总成一份完整报告，包括：

```text
测试概览
目标范围
资产发现
开放端口
漏洞总览
风险排序
验证过程
权限证明
证据索引
防守建议
工具版本
时间线
```

队员可以直接用作 writeup 初稿，裁判也可以据此复核过程。

## 五、蓝队队员使用流程

### 第一步：创建工作空间

进入平台后，点击：

```text
新建工作空间
```

填写：

```text
名称：例如 2026 蓝队演练第一场
说明：可选
```

工作空间相当于一次比赛或一次靶场任务的文件夹。

### 第二步：导入目标

支持三种方式：

```text
手动输入 IP
导入 IP/CIDR 列表
导入 URL/域名列表
```

示例：

```text
10.10.10.5
10.10.10.0/24
http://web01.range.local
https://target.example
```

导入后，平台会显示目标数量和目标类型。

### 第三步：选择扫描策略

平台提供三种策略：

| 策略 | 说明 | 适用场景 |
|---|---|---|
| 快速扫描 | 速度快，信息较少 | 比赛刚开始摸底 |
| 标准扫描 | 速度和深度平衡 | 推荐默认使用 |
| 深度扫描 | 更全面，耗时更长 | 目标较少时使用 |

建议比赛初期使用：

```text
标准扫描
```

如果时间紧张，可以先用：

```text
快速扫描
```

### 第四步：启动自动扫描

点击：

```text
开始扫描
```

平台会自动执行：

```text
存活探测
端口扫描
服务识别
Web 指纹识别
目录发现
漏洞模板扫描
```

扫描过程中可以实时查看：

```text
当前任务
已发现资产
开放端口
Web 服务
疑似漏洞
任务进度
```

### 第五步：查看漏洞排行

扫描结束后，进入：

```text
漏洞排行
```

平台会显示每个漏洞的：

```text
目标地址
端口服务
漏洞名称
风险等级
置信度
来源工具
推荐操作
```

队员重点关注：

```text
高风险
高置信度
管理后台
认证入口
文件上传
命令执行
未授权访问
弱配置
```

### 第六步：选择深度验证

勾选想要继续分析的漏洞，点击：

```text
深度验证
```

平台会进一步确认：

```text
漏洞是否真实存在
服务版本是否匹配
是否有可利用条件
是否存在误报可能
是否值得进入渗透验证
```

验证完成后，漏洞状态会变成：

```text
已确认
疑似误报
需要人工复核
验证失败
```

### 第七步：并行渗透验证

对于已经确认或高度疑似的漏洞，可以继续勾选，点击：

```text
并行渗透验证
```

平台会为每个任务创建独立临时容器。

页面会显示：

```text
任务编号
目标
漏洞名称
Runner 容器
执行状态
开始时间
运行时长
结果
证据
```

常见状态：

```text
排队中
运行中
证据采集中
已完成
失败
超时
已释放容器
```

### 第八步：查看证据

每个任务完成后，可以查看：

```text
结果摘要
截图
请求响应
日志
证明文件
时间线
```

证据用于：

```text
写 writeup
复盘攻击路径
向裁判证明过程
总结防守建议
```

### 第九步：生成整体报告

全部任务完成后，点击：

```text
生成报告
```

平台会自动生成：

```text
Markdown 报告
PDF 报告
证据索引
工具版本清单
```

队员可以在报告基础上补充自己的思路说明，形成最终 writeup。

### 第十步：结束项目并释放资源

点击：

```text
结束项目
```

平台会自动：

```text
停止未完成任务
收集剩余证据
释放临时容器
清理临时目录
标记项目完成
生成最终报告
```

## 六、裁判人员使用流程

裁判人员主要关注四类信息：

```text
目标是否符合比赛范围
任务执行是否规范
证据是否完整
容器是否释放
```

### 1. 查看工作空间

裁判可以进入指定工作空间，查看：

```text
导入目标
扫描时间
任务数量
漏洞数量
验证数量
报告状态
```

### 2. 查看任务审计

平台会记录每个关键动作：

```text
谁创建了任务
什么时候开始扫描
选择了哪些漏洞
启动了哪些验证容器
任务是否成功
证据是否生成
容器是否释放
```

### 3. 复核漏洞证据

裁判可以打开单个漏洞详情，查看：

```text
漏洞描述
发现来源
验证结果
证据截图
请求响应
工具输出
时间线
```

这可以帮助裁判判断：

```text
是否真实验证
是否存在误报
是否符合比赛要求
```

### 4. 查看容器释放情况

```text
创建时间
启动时间
结束时间
证据采集时间
释放时间
容器状态
```

裁判可以确认平台没有遗留运行中的任务容器。

### 5. 导出最终报告

裁判可以导出：

```text
整体渗透报告
任务执行日志
证据文件索引
容器生命周期记录
```

用于赛后评分、复盘和归档。

## 八、常见问题

### 1. 平台会自动攻击所有漏洞吗？

不会。

平台会自动发现和排序漏洞，但进入深度验证和渗透验证前，需要人工选择。

### 2. 为什么每个任务都要创建临时容器？

为了保证任务隔离。

这样可以避免：

```text
工具环境冲突
任务互相影响
临时文件残留
执行过程不可追踪
```

任务结束后，容器会自动释放。

### 3. 扫描结果很多，应该先看什么？

优先看：

```text
高风险
高置信度
管理后台
认证接口
Web 服务
数据库服务
文件上传点
未授权访问
远程命令执行类风险
```

平台会自动把这些内容排在前面。

### 4. 报告可以直接提交吗？

可以作为初稿，但建议队员补充自己的分析思路。

平台负责整理事实和证据，队员应补充：

```text
为什么选择这个目标
如何判断突破口
验证过程中遇到什么问题
最终学到了什么
```

### 5. 裁判如何判断任务是否真实执行？

裁判可以查看：

```text
任务日志
容器记录
工具输出
截图
请求响应
时间线
证据哈希
```

这些信息可以帮助复核。

## 九、使用建议

### 给蓝队队员

比赛开始后建议这样做：

```text
先快速导入全部目标
使用标准扫描摸清攻击面
优先查看漏洞排行前 10
先验证高置信度漏洞
多个目标并行验证
及时查看证据
最后统一生成报告
```

不要一开始就盲目深度扫描所有目标，容易浪费时间。

### 给裁判人员

建议重点关注：

```text
任务是否在比赛时间内执行
目标是否符合临时靶场要求
证据是否能支撑结论
是否存在异常高频扫描
容器是否全部释放
报告是否完整
```

## 十、平台价值总结

鹰翼外部靶场 AI 攻防工作台的核心价值可以概括为：

```text
让蓝队队员更快找到突破口
让渗透验证过程更规范
让多个漏洞验证可以并行执行
让证据自动沉淀
让 writeup 自动成型
让裁判复核更清楚
让赛后复盘更高效
```

它不是简单的扫描器，也不是自动攻击工具，而是一个面向比赛和教学的 **AI 辅助攻防演练工作台**。

最终目标是帮助蓝队队员在真实比赛节奏下，形成更系统的攻防思维、更规范的证据意识和更完整的复盘能力。
